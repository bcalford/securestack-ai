# v0.3-alpha release notes

SecureStack AI v0.3-alpha is a local-first defensive security review milestone focused on documentation clarity, repeatable validation, and end-to-end demo readiness. It does not add hosted scanning, private repository access, or production deployment automation.

## Highlights

- Real sample report page for demonstrating review output without requiring a fresh scan.
- One-command validation through `./scripts/validate-all.sh` and `make validate`.
- Duplicate-file and copy-artifact guardrails through `scripts/check-duplicates.sh`.
- Rule catalog exposed by the backend and frontend so reviewers can inspect deterministic local rule coverage.
- Public GitHub URL import for unauthenticated public repositories, analyzed locally with the same safety controls as uploaded files.
- Scan comparison from local scan history to show risk-score and finding deltas between two completed scans.
- Remediation workflow summary with finding status counts for open, reviewed, false-positive, and fixed findings.
- CI validation through GitHub Actions for backend, frontend, guardrail, secret-safety, and Docker Compose configuration checks.
- PDF and SARIF 2.1.0 export for completed local reviews.

## Validation expectations

Before sharing v0.3-alpha changes, run:

```bash
./scripts/validate-all.sh
docker compose -f docker-compose.yml -f docker-compose.postgres.yml config
```

If Docker is unavailable in the local environment, run `./scripts/validate-all.sh --skip-docker` and report the Docker limitation separately.

## Known limitations

- No authentication or authorization.
- No public deployment or hosted scanner.
- No private GitHub repository support, OAuth flow, GitHub App, or token-based import.
- No GitHub code scanning upload or automation.
- No OpenAI provider.
- No Semgrep execution or integration.
- SARIF export only; SARIF import/ingestion is not implemented.
- No multi-user production storage.
- No production AWS deployment automation.
